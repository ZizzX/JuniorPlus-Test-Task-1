🤖 Notes App AI Agent Workflow Rules (Task Tracking & Git Branching)

1. **Строгое следование плану**:
   - Агент ОБЯЗАН сверяться с файлами `plan.md` и `tasks.md` перед началом любой работы.
   - Запрещено перескакивать через задачи или брать в работу то, чего нет в `tasks.md` без явного разрешения пользователя.

2. **Работа с Git-ветками (Branching Strategy)**:
   - Прежде чем начать работу над новой задачей из `tasks.md`, агент ОБЯЗАН создать новую git-ветку.
   - Формат названия ветки: `feature/task-<номер_задачи>-<краткое_название>` (например, `feature/task-1.2-note-entity` или `fix/task-3.1-auth-bug`).
   - Вся работа по задаче ведется строго в этой ветке.

3. **Обновление статуса задач (Task Tracking)**:
   - Когда задача взята в работу, агент должен изменить ее статус в `tasks.md` на: `🔄 IN PROGRESS (Branch: <название_ветки>)`.
   - По завершению задачи (после успешных тестов и линтинга), агент ОБЯЗАН обновить `tasks.md`, пометив задачу как завершенную.
   - Формат завершенной задачи: `- [x] Название задачи - ✅ DONE (Date: YYYY-MM-DD, Branch: <название_ветки>)`.
   - Если по ходу выполнения задачи были приняты важные архитектурные решения или возникли нюансы, агент должен добавить краткое пояснение `> Примечание: ...` под выполненным пунктом.

4. **Завершение и коммиты**:
   - По завершению задачи агент должен предложить понятный commit message в стиле Conventional Commits.
   - После фиксации коммита агент запрашивает разрешение на переход к следующей задаче.

5. Согласно правилам хорошего тона и для избежания конфликтов, завершенные и протестированные задачи следует сливать в основную ветку (`main`), чтобы она всегда содержала актуальное и рабочее состояние проекта.
